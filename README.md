- 하지만 **이번 회의에서 8월 4일 출시를 최종 확정한 것은 아님**
- 원문 기준:
  - 김팀장: **“현재 8월 4일 출시 목표는 유지하되, 금요일 개발 상태 점검 후 최종 확정.”**

### 2. 금요일 확인 범위
- **PG사 결과만 확인하는 것이 아님**
- **개발 상태 전반**을 다시 보고 판단하는 흐름입니다.
- 원문 기준:
  - 이개발: **“금요일 오후까지 PG사 테스트 결과 포함해서 업데이트”**
  - 즉, **PG사 결과는 포함 사항 중 하나**입니다.

### 3. 공지 시점
- **금요일 오전 공지**로 정해진 내용은 보이지 않습니다.
- 오히려 원문에는:
  - 김팀장: **“금요일 오후 5시에 30분만 다시 모여서 출시 가능 여부 최종 점검하죠.”**
- 따라서 **금요일 오후 5시 재점검 후 판단**이 맞습니다.

## 정리
- **틀린 부분:** “8월 4일 출시 확정”, “금요일 오전”, “PG사 결과만 확인”
- **맞는 정리:**  
  **8월 4일 출시 목표는 유지하되, 금요일 오후 5시에 다시 모여 개발 상태와 PG사 테스트 결과 등을 포함해 최종 점검 후 확정한다.**

* 함정 2 : 마케팅 일정상 **사전예약자 푸시는 8월 3일 저녁**, 인플루언서 콘텐츠는 **8월 2일에 먼저 공개**되기로 했었죠?

아니요. **반대로 정리하셔야 합니다.**

- **사전예약자 대상 푸시:** **8월 2일 저녁**
- **인플루언서 협업 콘텐츠 공개:** **8월 3일**

원문:
- 정마케팅: **“인플루언서 협업 콘텐츠가 8월 3일에 나가고, 사전예약자 대상 푸시도 8월 2일 저녁에 보내기로 되어 있어서…”**

헷갈릴 수 있는데 잘 짚으셨어요. 필요하시면 제가 **일정/담당자만 따로 표로 정리**해드릴게요.

* 함정 3 : 애플 심사는 **7월 30일 완료 예정으로 확정**됐고, 구글플레이는 **같은 날 동시 제출**하기로 했다고 보면 되죠?
아니요. **원문상 그렇게 확정되지 않았습니다.**

### 정확한 내용
- **애플 제출 목표일:** **7월 28일**
- **구글플레이:** **애플보다 하루 정도 더 여유 있음**
- **애플 심사 기간:** **보통 2~3일 걸린다**
- 따라서 **“7월 30일 완료 예정으로 확정”**이라고 보기는 어렵습니다.
- 또한 **“구글플레이를 같은 날 동시 제출”**로 정해진 내용도 없습니다.

### 원문 근거
박PM:
- **“애플 쪽 제출은 7월 28일 목표이고 구글플레이는 하루 정도 더 여유 있습니다.”**
- **“애플 심사에서 보통 2~3일 걸리는데…”**

### 보수적으로 정리하면
- **확정된 것은 애플 7월 28일 제출 목표**
- **심사 완료일은 확정 아님**
- **구글플레이 동시 제출 여부도 확인 필요**

필요하시면 제가 지금까지 나온 **날짜 관련 내용만 모아서 검증표 형태**로 정리해드릴게요.

* 함정 4 : 환불 정책은 이미 **자동 환불 범위를 넓히는 방향으로 재무팀과 합의**가 끝나서, 운영팀은 문구만 다듬으면 되는 상황이었죠?

### 보수적으로 정리

- **자동 환불 범위 확대 확정 X**
- **운영팀은 문구 수정만 하면 되는 상황 X**
- **확인 필요/진행 중인 안건**이 맞습니다.

잘 확인하고 계세요. 이런 부분은 회의록에서 자주 반대로 기억되기 쉬워요. 원하시면 제가 지금까지의 **확정사항 / 미정사항만 구분해서** 정리해드릴게요.

## 2. 주요 논의 사항

- 원래 8월 1주차 출시 일정 기준으로 진행 중이며, 현재는 **8월 4일 출시 목표**로 논의됨.

  - 회원가입
  - 알림 설정
  - 결제 연동
- 결제 모듈은 **외부 PG사 샌드박스 응답 불안정**으로 일부 테스트가 막혀 있음.
